    • Thoughts on Progressive Enhancement

    A List Apart's 'Understanding Progressive Enhancement' was written by Aaron Gustafson in 2008. This article recounts the journey of the web development approach from graceful degradation – an idea borrowed from engineering whereby a site is first developed for the most modern, fully-featured, accessible browsers and is expected to 'gracefully degrade' in older or simpler browsers – to progressive enhancement. The main thrust of Gustafson's argument is that this top-down approach has been wrong all along. The more suitable philosophy is progressive enhancement, a term coined at SXSW in 2003 by Steve Champeon and Nick Finck, with content rather than the browser as its focal point. The most famous metaphor used in the article is the Peanut M&M analogy; you start with the peanut, which represents the content, then coat it with a layer of CSS and a final layer of the 'hard candy shell' which is the JavaScript to add the fully interactive experience. Each layer can be seen as an optional progressive enhancement - a user may only see the peanut, another the peanut and the chocolate (the CSS) or even the whole Peanut M&M. Graceful degradation concerns itself with the browser, progressive enhancement concerns itself with content, the reason for a website to exist.

    Progressive enhancement has an immense importance and arguably increased importance in the modern web. Essentially, progressive enhancement is an approach that takes a "bottom-up" approach instead of a "top-down" approach, beginning with content that is functional regardless of environment and then progressively layering in presentational elements through CSS and interactivity through JavaScript. It is for these reasons that the dependency on the JavaScript layer is considered problematic, given that errors from networks, extensions, or browser settings can result in a website that isn't functional, hence, the foundation is brittle. Accessibility is another advantage; this will still be achievable with semantic HTML even if neither the CSS nor the JavaScript works properly, which in many cases is a legal requirement as well as a good development practice. Performance is also a significant aspect of the user's experience when dealing with the vast number of various devices that access the web. There are no restrictions in what you may be attempting to build; it's instead a guarantee. Progressive enhancement underlies the concept of modern design including Mobile First, Progressive Web Apps and performance optimization.

    Mars Rover Project Revision and Reflection

    When I first made my draft I started really late, and ultimately decided to just leave my unfinished product as is with the FAQ and History pages unfinished. Since it was a draft assignment I knew I shouldn't really be worrying about completion, and more having a great foundation for the website structurally. The final project was just me envisioning what a NASA homepage would look like as well as referencing the actual NASA page for Mars exploration as inspiration. I feel proud of the way my website came out — the design is basic but everything functions as intended and I didn't encounter many errors in my work process.
